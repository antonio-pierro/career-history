Monitoring and accounting operation  1/2

  • Goal: provide precise and reliable information about the status and usage of Grid resources
  • Challenge: dynamic and heterogeneous infrastructure, sensitive data, different users (many experiments exploit the Grid)
  • Solution: a monitoring tool using personal digital certificate to recognize users' role

Monitoring and accounting operation  2/2

  • Site administrator: how many jobs submitted on my site are running or queued?
  • VO Manager: how many jobs submitted by my users are running or queued? On which site are running them?
  • Grid users: what is the status of my jobs on the Grid?
  • Site Administrator: is there any service down?

Handling of CMS condition data 1/2

  • Goal: handle all condition data coming from offline calibration
  • Challenge: crucial for the CMS experiment; two different network layers which are separated by a firewall
  • Solution: tool for automatic upload in Oracle Database (traceability, security copy, online network polls GPN)

Handling of CMS condition data 2/2

Monitoring of CMS condition data 1/2

  • Goal:
    • Monitor the population of CMS condition database accounts in real-time
    • Spot problematic dataset that could block the express data processing
  • Challenge:
    • Integration in CMSSW standards
    • Usable Interface
    • Ease of implementation and maintenance
  • Solution: web-based monitoring tool using the latest web technologies (persistent local storage, Web-Socket, Web-Workers) and the 3-tier architecture.
